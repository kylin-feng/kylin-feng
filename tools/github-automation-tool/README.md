# GitHub 自动化工具

一个合法合规的GitHub活跃度维护工具，通过自动生成有意义的学习内容来保持仓库活跃。

## 功能特性

### 🎯 核心功能
- **日常学习日志**: 自动生成每日技术学习记录
- **学习笔记生成**: 智能创建各类技术学习笔记
- **文档自动更新**: 定期更新README和项目统计
- **定时任务调度**: 每天晚上8点自动执行任务

### 🌟 特色亮点
- **合法合规**: 生成真实有意义的学习内容，不是虚假提交
- **智能随机**: 随机选择任务类型，模拟真实的学习节奏
- **Web管理界面**: 简洁美观的配置和管理页面
- **灵活配置**: 支持自定义仓库路径、分支等参数

## 快速开始

### 环境要求
- Python 3.7+
- Git
- 一个已初始化的GitHub仓库

### 安装步骤

1. **克隆或下载项目**
```bash
git clone <项目地址>
cd github-automation-tool
```

2. **安装依赖**
```bash
pip install -r requirements.txt
```

3. **启动应用**
```bash
python app.py
```

4. **访问Web界面**
打开浏览器访问: http://localhost:58899

### 配置设置

1. **配置仓库路径**
   - 进入配置页面
   - 输入你的GitHub仓库本地路径
   - 设置远程仓库名称（通常是origin）
   - 设置目标分支（通常是main）

2. **测试连接**
   - 点击"测试仓库连接"确保配置正确
   - 确保本地仓库已正确关联远程GitHub仓库

3. **启动自动化**
   - 应用启动后自动开始定时任务调度
   - 每天晚上8点执行自动化任务

## 工作原理

### 自动生成内容类型

1. **日常学习日志** (`daily_logs/`)
   - 记录每日学习任务和技术要点
   - 包含时间戳和学习统计
   - Markdown格式，便于阅读

2. **技术学习笔记** (`learning_notes/`)
   - 随机选择技术主题
   - 包含代码示例和最佳实践
   - 涵盖多个编程领域

3. **README更新**
   - 自动更新项目统计信息
   - 显示最后更新时间
   - 维护项目结构说明

### 安全特性
- 本地Git操作，不直接推送到远程
- 生成真实有价值的学习内容
- 支持手动触发和自动定时执行
- 完整的错误处理和日志记录

## Web界面功能

### 主页面
- 实时显示任务状态
- 仓库连接状态检查
- 手动执行任务按钮
- 功能特性展示

### 配置页面
- 仓库路径设置
- Git参数配置
- 连接测试功能
- 配置说明和帮助

### API接口
- `/api/run_task` - 手动执行任务
- `/api/test_repo` - 测试仓库连接

## 使用建议

1. **选择合适的仓库**: 建议为学习记录创建专门的仓库
2. **定期检查**: 偶尔查看生成的内容，确保质量
3. **手动补充**: 可以手动添加更多个性化的学习内容
4. **备份配置**: 重要的配置信息建议备份

## 注意事项

- 确保Git仓库已正确配置用户信息
- 建议在测试仓库中先试用
- 生成的内容旨在记录真实的学习轨迹
- 请根据个人需要调整执行频率
