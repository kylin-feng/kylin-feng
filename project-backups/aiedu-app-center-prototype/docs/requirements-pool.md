# AI教育平台需求池与迭代分级

整理日期：2026-05-19

## 1. 分级规则

| 级别 | 含义 | 判断标准 |
|---|---|---|
| P0 | 首版必须做 | 高频教师场景、能形成教学闭环、可快速验证价值 |
| P1 | 第二阶段增强 | 依赖 P0 数据或流程，能提升留存和复用 |
| P2 | 场景扩展 | 面向学科特色、课堂互动、AI实验项目 |
| P3 | 探索储备 | 创新性强但复杂度、合规或运营成本较高 |

## 2. 建议迭代节奏

| 版本 | 目标 | 重点能力 |
|---|---|---|
| V0.1 MVP | 先跑通教师日常工作流 | 备课、教案、PPT、出题、OCR、作业批改 |
| V0.2 评价闭环 | 把作业/考试结果转成报告和改进建议 | 教学周报、评估报告、因材施教、班级数据 |
| V0.3 学科扩展 | 做出学科应用和课堂互动特色 | 学科智能体、实验室、知识图谱、项目设计 |
| V0.4 平台生态 | 支持更多角色、作品、赛事和智能体配置 | 数字人、AI评审、自动化、创作类应用 |

## 3. P0 首版核心需求

| 需求 | 对应应用 | 用户价值 | 备注 |
|---|---|---|---|
| AI备课工作台 | AI备课、教案优化、课程规划 | 老师输入课标、年级、教材、课时，生成教学目标、重难点、流程、活动设计 | 建议先合并为一个“备课助手”，不要拆太细 |
| PPT生成 | AI-PPT | 从教案/大纲生成课件结构和页面内容 | 首版可先做大纲到PPT草稿 |
| 出题与组卷 | 智能组卷、AI出题、数学出题助手 | 按年级、学科、知识点、题型、难度生成题目和答案解析 | “智能组卷”和“AI出题”应共用题库/生成能力 |
| OCR资料识别 | OCR识别 | 图片、试卷、作业转文本，作为后续批改和分析输入 | 是作业批改和评估报告的基础能力 |
| 作业批改 | 作业批改 | 自动批改客观题，辅助评价主观题，输出错因和建议 | 首版先限定题型和学科，降低风险 |
| 教学资源保存与导出 | 备课、PPT、组卷、批改共用 | 生成内容能保存、编辑、复制、导出 | 这是所有生成类应用的底座 |

## 4. P1 评价与班级运营需求

| 需求 | 对应应用 | 用户价值 | 依赖 |
|---|---|---|---|
| 教学周报 | 教学周报 | 汇总本周课程、作业、学生表现，生成周报 | 课程记录、作业批改数据 |
| 评估报告 | 评估报告 | 基于作业/考试生成学生或班级诊断报告 | OCR、批改、成绩数据 |
| 因材施教 | 因材施教专家 | 按学生层次推荐内容、作业和练习 | 评估报告、知识点标签 |
| 课班数据看板 | 课班数据仓、动态图表 | 展示作业完成率、正确率、知识点掌握情况 | 数据采集与可视化 |
| 知识点结构化 | 知识图谱、思维导图 | 把课程内容、错题、知识点串起来 | 题目和教案的知识点标注 |

## 5. P2 学科应用与课堂互动需求

| 方向 | 对应应用 | 建议处理方式 |
|---|---|---|
| 语文学习 | 作文启迪大师、古诗问答、经典解析、成语、绕口令、谜语 | 先做成“语文智能体包”，统一入口 |
| 英语学习 | 英语陪练、万能翻译 | 先做口语陪练和句子纠错，翻译作为工具能力 |
| 理科实验 | 光学实验室、力学实验室、电学实验室、生物实验室 | 适合做课堂演示组件，和课程资源绑定 |
| AI实验室 | 手势识别、图像识别、语音识别、水果分类、姿态识别、文本识别 | 偏项目式学习，放在第二批或校本课程中 |
| 项目化学习 | 项目设计、学科融合课例、编程助手 | 适合作为教师备课增强，不建议首版独立做大 |

## 6. P3 探索储备需求

| 需求 | 对应应用 | 暂缓原因 |
|---|---|---|
| 数字人老师 | 数字人老师 | 制作成本、形象授权、内容审核复杂 |
| AI自动化专家 | AI自动化专家 | 需要接入多个平台流程，前期边界不清 |
| AI评审 | AI评审 | 评分公平性、规则解释、申诉流程需要设计 |
| 心理导师 | 心理导师 | 涉及心理健康与安全合规，需要非常谨慎 |
| 手语/无障碍助手 | 手语翻译、语音学习助手 | 价值高，但识别准确率和适配成本较高 |
| 创意生成 | 绘本生成、游戏设计大师、剧本大师、学科视频制作、幽默大师 | 适合做后续创作生态，首版不是刚需 |
| 生活化AI示例 | 服装推荐、房价预测、决策树 | 可作为AI课程案例，不建议作为核心教学功能 |

## 7. 推荐首批应用清单

首批建议聚焦 6 个入口：

1. 备课助手：覆盖 AI备课、教案优化、课程规划。
2. AI-PPT：从教案或主题生成课件草稿。
3. 出题组卷：覆盖智能组卷、AI出题、数学出题助手。
4. OCR识别：上传图片/试卷/资料，转成可编辑文本。
5. 作业批改：基于 OCR 和答案规则完成批改与错因分析。
6. 教学周报：把备课、作业、评价数据汇总成周报。

## 8. 下一步需要确认的问题

| 问题 | 为什么重要 |
|---|---|
| 首批服务对象是老师、学生还是教研员？ | 决定首页、权限和核心流程 |
| 首批覆盖哪些学段和学科？ | 决定题型、教材、课标、提示词模板 |
| 是否必须支持导出 Word/PPT/PDF？ | 影响技术选型和交付验收 |
| 作业批改先做客观题还是也做主观题？ | 影响准确率、人工复核和风险边界 |
| 是否有学校现成数据源？ | 影响周报、评估报告、数据看板能不能首版落地 |
